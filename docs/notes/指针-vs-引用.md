# 指针 vs 引用

指针（pointer）和引用（reference）都是 C++ 中"间接访问对象"的机制，但它们在语义、使用约束和典型场景上有清晰的分工。面试中被问"指针和引用有什么区别"，答"指针可为空、引用不行"是起点；真正能区分中级和高级的，是能否讲清**内存模型的差异、const 修饰的层次、底层实现与抽象语义的分离**，以及**何时必须用哪个**。

## 简要回答

**指针**是**存储地址的独立变量**：可重新赋值、可为空、支持算术运算，但需要手动管理内存（特别是所有权）。**引用**是**已存在对象的别名**：必须在声明时初始化、不能为空、绑定后终身不可更改，语法上与原对象完全等价。

一句话区分：**指针是"我知道他住哪"，引用是"我就是他的另一个名字"**。

## 维度对比

| 维度 | 指针 | 引用 |
|---|---|---|
| 本质 | 存储地址的独立变量 | 已存在对象的别名 |
| 内存占用 | 独立占用 4/8 字节（取决于平台） | 无独立存储（编译器内部可能用指针实现） |
| 初始化 | 可延迟初始化 | **必须在声明时初始化** |
| 空值 | 可为 `nullptr` | **不能为空** |
| 重新绑定 | `p = &b` 随时可改指向 | 绑定后终身不变 |
| 多级间接 | 支持 `int**`（指针的指针） | 不存在"引用的引用" |
| 算术运算 | 支持 `p++`、`p + n` | 不支持 |
| 访问目标 | 需解引用 `*p` | 直接使用，语法透明 |
| `sizeof` | 返回指针本身大小（4/8） | 返回绑定对象的大小 |
| const 修饰 | 多种组合（见下文） | 没有"const 引用"的概念（`const T&` 修饰的是被引用对象） |

## 关键原理

### 内存模型差异

指针是**一级公民**——它本身是一个变量，存储了另一个对象的地址，因此有自己的内存、可以单独操作（取地址 `&p`、解引用 `*p`、算术运算）。

引用是**语法糖**——`int& r = a;` 之后，`r` 和 `a` 在源代码层面完全等价（同一个对象的不同名字）。**编译器在底层通常用指针实现引用**（因为 CPU 没有"别名"概念），但**抽象语义层不暴露这一点**——你不能对引用取地址后重新指向别处。

```cpp
int a = 10;
int* p = &a;  // p 存了 a 的地址
int& r = a;   // r 是 a 的别名，编译器可能让 r 内部就是 &a
```

### const 修饰的层次

指针有**三种 const 组合**：

| 写法 | 含义 | 能否改值 | 能否改指向 |
|---|---|---|---|
| `const int* p`（或 `int const* p`） | 指向 const 值的指针 | ❌ | ✅ |
| `int* const p` | const 指针（指针本身不能改） | ✅ | ❌ |
| `const int* const p` | 双 const | ❌ | ❌ |

**口诀**：**const 在 `*` 左边修饰指向的值，在 `*` 右边修饰指针本身**。

引用**没有自己的 const 概念**——`const int& r = a;` 中 const 修饰的是**被引用的对象**（承诺不通过 r 改 a），而不是引用本身（引用本来就不能改绑定）。所以"const 引用"实际是"绑定到 const 对象的引用"。

### 实现 vs 语义的分离

这是理解 C++ 抽象能力的关键：

- **运行时**：引用和指针生成的机器码几乎一致（都是间接寻址）
- **编译期**：编译器对引用施加了**额外约束**（必须初始化、不能为空、不能重绑），违反会直接编译失败
- **源码层**：引用是"透明别名"，写起来像直接操作原对象

这种"实现用一种机制，语义暴露另一种抽象"是 C++ 设计的核心模式。**理解它**才能理解为什么 `Base& obj = derived_obj;` 能直接走派生类的 vtable（语义是"obj 是 Base 类型"），而 `Base* p = &derived_obj;` 也行（语义是"p 指向一个 Base"）——两者在汇编层可能一样，但语义层赋予了不同的承诺。

## 选择指南

**用指针的场景**（三种）：

1. **可能为空**：可选参数、找不到目标、错误码替代
2. **需要改变指向**：链表遍历、容器迭代、`vector::push_back` 内部
3. **表达"无主"或所有权**：智能指针、工厂模式返回

**用引用的场景**（大多数）：

- **函数参数**：避免拷贝（`void f(const T&)`）
- **函数返回值**：链式调用（`std::ostream& operator<<(...)`）
- **范围 for 循环**：`for (const auto& x : container)`
- **多态基类**：`Base& obj = derived;`
- **运算符重载**：`T& operator=()`

**现代 C++ 准则**：

- 能用引用就用引用——语法干净、不用判空、不会野指针
- 需要所有权语义时用智能指针（`unique_ptr` / `shared_ptr`），不要用裸指针
- 裸指针**只用于"非拥有观察"**（即你只借来用，不负责释放）

### 函数参数选择

```cpp
// 必传：用 const 引用（高效 + 安全 + 表达"不修改"）
void print(const std::string& s);

// 可能不传：用指针（nullptr 表示"无"）
void print_or_skip(const std::string* s);

// 必传且要修改：用引用（隐含承诺"会修改"）
void append_space(std::string& s);
```

参数选择依据是**语义**而非**性能**——三者生成的机器码几乎一样（都是传地址），但传递给读者的承诺不同。
