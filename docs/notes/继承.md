# 继承

继承（Inheritance）是 [C++ 学习路线(2026)](./c++-学习路线2026.md) 之一，允许派生类自动复用基类的成员，并在基类行为之上扩展或修改。继承的本质是**类型之间的"is-a"关系**与代码复用机制的结合，但只有当类型之间真的满足 is-a 语义时，继承才合适——单纯为了省事复用代码而继承，往往是设计腐化的起点。

## 定义与核心思想

继承回答两个问题：

- **类型层面**：派生类是基类的一种特化（"Dog is-a Animal"），因此可以出现在所有期望基类的地方。
- **实现层面**：派生类自动获得基类的非 `private` 成员，无须重新实现。

C++ 支持三种继承方式：`public` / `protected` / `private`，区别在于基类成员在派生类中的访问级别如何变化。

## 关键原理

### 三种继承方式

| 继承方式 | 基类 `public` 成员 | 基类 `protected` 成员 | 基类 `private` 成员 | 典型用途 |
|---|---|---|---|---|
| `public` | 派生类中仍为 `public` | 派生类中仍为 `protected` | 不可见 | 表达 is-a（最常用） |
| `protected` | 派生类中变为 `protected` | 派生类中仍为 `protected` | 不可见 | 几乎不用 |
| `private` | 派生类中变为 `private` | 派生类中变为 `private` | 不可见 | 实现"has-a"借用实现（罕见） |

`class` 默认 `private` 继承，`struct` 默认 `public` 继承。**实际项目里 99% 的场景都是 `public` 继承**，其他两种方式通常意味着"我想复用基类实现但又不想让它成为接口的一部分"——这种需求在现代 C++ 里通常用 [继承](./继承.md) 或 [C++ 多态的实现机制](./c++-多态的实现机制.md) 解决。

### override 与 [虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md)

派生类可以**重写**（override）基类的虚函数，从而改变该方法在派生类语境下的行为。这是 [多态](./多态.md) 的实现基础：

```cpp
class Animal {
public:
  virtual std::string speak() const { return "..."; }
  virtual ~Animal() = default;
};

class Dog : public Animal {
public:
  std::string speak() const override { return "Woof"; }
};
```

注意两点：

1. `override` 是 C++11 引入的**显式标记**，让编译器帮你检查"真的在重写某个虚函数"，避免签名写错导致意外隐藏。
2. 任何含虚函数的类都应有**虚析构函数**——否则通过基类指针 `delete` 派生类对象时，派生类的析构函数不会被调用，资源泄漏。

### 继承的语义边界

继承只对"is-a"成立有意义：

- ✅ `Dog : public Animal` —— Dog 是 Animal 的一种
- ❌ `Square : public Rectangle` —— 正方形是矩形吗？数学上 yes，但 Rectangle 的"宽高可独立修改"在 Square 上不成立，继承后基类的 `setWidth` 会破坏 Square 的不变式
- ❌ `Stack : public Vector` —— 复用底层存储但暴露了 Vector 的所有接口，破坏 Stack 的 LIFO 不变式

对这些"实现复用但语义不同"的场景，应当使用 [继承](./继承.md)（has-a）或私有继承。

## 应用与实例

### 最小可运行示例

```cpp
class Shape {
public:
  virtual double area() const = 0;
  virtual ~Shape() = default;
};

class Circle : public Shape {
public:
  Circle(double r) : r_(r) {}
  double area() const override { return 3.14159265 * r_ * r_; }
private:
  double r_;
};

class Rectangle : public Shape {
public:
  Rectangle(double w, double h) : w_(w), h_(h) {}
  double area() const override { return w_ * h_; }
private:
  double w_, h_;
};
```

`Circle` 和 `Rectangle` 都"是 `Shape`"，都可以参与 `std::vector<std::unique_ptr<Shape>>` 的存储与多态调用。

### 典型使用场景

- **领域建模**：表达类型层级（如 `Employee` → `Manager` → `Engineer`）
- **框架扩展点**：框架定义虚函数基类，用户继承并重写
- **设计模式**：[Strategy]、[Template Method]、[State] 都依赖继承 + 虚函数

## 与其他概念的关系

- [封装](./封装.md) 在继承体系中仍然有效：派生类对外部是封装好的对象，但对基类而言其 `protected` 成员是可见的——封装粒度按"类"而非"对象"划分。
- [多态](./多态.md) 必须依赖继承 + 虚函数才能实现运行时多态。**没有虚函数的继承不产生多态，只产生代码复用**。
- [C++ 学习路线(2026)](./c++-学习路线2026.md) 三个概念中，继承是"代码复用"与"接口扩展"的双重机制；[继承](./继承.md) 则是替代继承实现复用的另一条路。
- [虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md) 决定了继承层级对外暴露哪些接口；[虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md) 决定了派生类能修改哪些行为。
