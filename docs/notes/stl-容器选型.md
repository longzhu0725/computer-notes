# C++ STL 容器选型

## 核心结论

STL 容器分**三大类 + 一类适配器**:**顺序容器**(按插入顺序)、**关联容器**(有序,红黑树)、**无序容器**(无序,哈希表)、**容器适配器**(接口封装)。**选型的本质是底层数据结构的选择**——连续内存(缓存友好但失效严重)还是离散节点(插入稳定但局部性差)。**默认用 `vector`**,需要时才考虑其他。

| 类别 | 容器 | 底层结构 | 适用场景 |
|------|------|---------|---------|
| 顺序 | `vector` | 动态数组(连续) | 90% 场景默认 |
| 顺序 | `deque` | 分段连续 | 两端频繁增删 |
| 顺序 | `list`/`forward_list` | 双向/单向链表 | 中间频繁增删 + 迭代器稳定 |
| 顺序 | `array` | 编译期固定数组 | 替代 C 数组,零开销 |
| 关联 | `map`/`set` | 红黑树 | 有序 + 范围查询 |
| 关联 | `multimap`/`multiset` | 红黑树 | 允许重复键 |
| 无序 | `unordered_map`/`unordered_set` | 哈希表 | 仅需快速查找 |
| 无序 | `unordered_multimap`/`unordered_multiset` | 哈希表 | 允许重复键 + 快速查找 |
| 适配器 | `stack` | 默认 `deque` | LIFO |
| 适配器 | `queue` | 默认 `deque` | FIFO |
| 适配器 | `priority_queue` | 默认 `vector` + 堆 | 优先级排序 |

## 一、顺序容器详解

### `vector`:默认之王

```cpp
std::vector<int> v = {1, 2, 3, 4, 5};
// 优点:
//   - 随机访问 O(1)
//   - 尾部插入均摊 O(1)(偶尔扩容 O(n))
//   - 缓存友好(连续内存)
//   - 与 C API 兼容(data() 拿裸指针)
//
// 缺点:
//   - 中间插入 O(n)(后续元素后移)
//   - 扩容时所有迭代器失效
//
// 适用:
//   - 90% 的场景
//   - 不确定用什么就用它
```

**最佳实践**:**预分配**避免扩容:
```cpp
std::vector<int> v;
v.reserve(1000);  // 预分配 1000 容量,避免后续 N 次重新分配
for (int i = 0; i < 1000; ++i) v.push_back(i);
```

### `deque`:两端操作

```cpp
std::deque<int> dq;
// 优点:两端 push/pop O(1)
// 缺点:
//   - 不像 vector 那么缓存友好(分段连续)
//   - 不能用 data() 拿裸指针
//   - 中间插入仍然 O(n)
//
// 适用:双端队列、滑动窗口、消息队列
```

**实战经验**:**绝大多数场景用 `vector` 代替 `deque`**,只有当 `push_front()` 性能关键时才用 `deque`。

### `list` / `forward_list`:现代 C++ 很少用

```cpp
std::list<int> lst;  // 双向链表
std::forward_list<int> fl;  // 单向链表
// 优点:
//   - 任意位置插入/删除 O(1)(已知迭代器)
//   - 插入/删除只失效当前迭代器,其他不受影响
// 缺点:
//   - 缓存不友好(节点离散)
//   - 不支持随机访问
//   - 内存开销大(每个节点 2-3 个指针)
//
// 适用:极少!
//   - 中间插入/删除极频繁 + 持有迭代器
//   - 大对象 + 频繁移动(避免 vector 中间删除的 O(n) 拷贝)
```

**实战经验**:**现代 C++ 优先用 `vector`**——除非有明确证据 `list` 更快,否则不要用。

### `array`:零开销固定数组

```cpp
std::array<int, 5> arr = {1, 2, 3, 4, 5};
// 编译期大小,无运行时分配,sizeof = sizeof(int) * 5
// 替代 C 数组 arr[5] 的安全版本
//
// 适用:
//   - 知道大小的固定集合
//   - 栈上分配,无堆开销
//   - 与 C API 互操作
```

## 二、关联容器详解

### `map` / `set`:有序

```cpp
std::map<std::string, int> m;
m["apple"] = 1;
m["banana"] = 2;
// 优点:键自动按字典序排列
// 范围查询:m.lower_bound("b"), m.upper_bound("c")
// 适用:需要有序遍历 / 范围查询
```

**对比 `set` vs `map`**:`set` 只存键,常用于"判断元素是否存在 + 去重";`map` 存键值对。

### `multimap` / `multiset`:允许重复键

```cpp
std::multimap<int, std::string> mm;
mm.insert({1, "a"});
mm.insert({1, "b"});  // 允许 key=1 重复
// 适用:一对多关系(如学生-科目成绩)
```

## 三、无序容器详解

### `unordered_map` / `unordered_set`

```cpp
std::unordered_map<std::string, int> um;
um["apple"] = 1;
// 平均 O(1) 查找/插入/删除
// 无序
// 适用:只需要快速查找,不需要有序
```

**与 `map` 选型**:
- 需要有序 → `map`
- 只需查找 → `unordered_map`(快 10-20 倍)
- 详细对比见 [map vs unordered_map](./map-vs-unordered_map.md)

## 四、容器适配器

### `stack` / `queue`:接口封装

```cpp
std::stack<int> st;  // 默认底层 deque
std::queue<int> q;   // 默认底层 deque
// 不是独立数据结构,只是限制了底层容器的接口
// 暴露 push/pop/top,不允许随机访问
```

### `priority_queue`:堆

```cpp
std::priority_queue<int> pq;  // 默认底层 vector,实现大顶堆
pq.push(3);
pq.push(1);
pq.push(4);
pq.top();  // 4(最大元素)
pq.pop();  // 弹出 4
// 适用:Top K 问题、Dijkstra 最短路、任务调度
```

## 五、快速选型决策表

| 需求 | 推荐容器 |
|------|---------|
| 随机访问 + 尾部增删 | `vector` |
| 两端频繁增删 | `deque` |
| 中间频繁增删(已知位置) | `list` |
| 有序键值查找 + 范围查询 | `map` |
| 快速查找(不需要有序) | `unordered_map` |
| 去重 + 有序 | `set` |
| 去重 + 快速判断存在 | `unordered_set` |
| 固定大小 + 栈分配 | `array` |
| LIFO | `stack` |
| FIFO | `queue` |
| 优先级排序 / Top K | `priority_queue` |

## 六、内存布局对比

```
vector:  ┌───┬───┬───┬───┬───┐         ← 连续内存,缓存友好
         │ 1 │ 2 │ 3 │ 4 │ 5 │
         └───┴───┴───┴───┴───┘
         ↑ data() 拿裸指针 ✓

deque:   ┌───┐ ┌───┐ ┌───┐              ← 分段连续
         │ A │ │ B │ │ C │              (每段是连续的小数组)
         └─┬─┘ └─┬─┘ └─┬─┘
           ↓     ↓     ↓
         ┌───┐┌───┐┌───┐┌───┐┌───┐
         │ 1 ││ 2 ││ 3 ││ 4 ││ 5 │
         └───┘└───┘└───┘└───┘└───┘

list:    ┌───┐ → ┌───┐ → ┌───┐ → ...     ← 离散节点
         │ 1 │   │ 2 │   │ 3 │
         └─┬─┘   └─┬─┘   └─┬─┘
           ↓       ↓       ↓
         (堆上)   (堆上)   (堆上)        ← 缓存不友好
```

## 七、面试高频追问

**Q1: 默认用哪个容器?**
**`vector`**。连续内存 + 缓存友好 + 随机访问 O(1) + 尾部插入均摊 O(1)。90% 场景都适用。**只有明确不适用时**(频繁中间插入、要求迭代器稳定)才考虑其他。

**Q2: vector 和 deque 怎么选?**
默认 `vector`,除非需要频繁 `push_front()`。vector 内存完全连续,缓存性能最好,且可用 `data()` 拿裸指针传给 C API。deque 分段连续,做不到。

**Q3: map 和 unordered_map 怎么选?**
- **需要有序**(范围查询、按序遍历)→ `map`
- **只需快速查找** → `unordered_map`
- 数据量小时 `map` 可能更快(红黑树常数小);数据量大时 `unordered_map` 优势明显

**Q4: 什么时候用 list?**
**现代 C++ 中极少**。只有两种情况值得用:
1. 中间频繁插入/删除,且已持有迭代器
2. 大对象,vector 移动 O(n) 拷贝代价高
其他场景 `vector` 几乎总是更快(缓存局部性优势太大)。

**Q5: 容器适配器和容器有什么区别?**
适配器**不是独立的数据结构**,只是限制了底层容器的接口。`stack` 禁止随机访问只暴露 `push/pop/top`,底层默认是 `deque` 但可换成 `vector` 或 `list`。适配器的意义是**语义约束**——告诉使用者"这里只能后进先出"。
