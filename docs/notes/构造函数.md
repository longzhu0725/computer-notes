# C++ 构造函数六种类型

## 核心结论

C++ 构造函数有**六种**:默认、参数化、拷贝、移动、委托、继承。前四种解决"怎么创建对象",后两种解决"怎么复用初始化逻辑"。**移动构造**是 C++11 性能优化的核心手段——面试必问。

| 类型 | 形式 | 编译器自动生成? | 关键作用 |
|------|------|---------------|---------|
| 默认构造 | `ClassName()` | 未定义任何构造函数时 | 零初始化 |
| 参数化构造 | `ClassName(int x, ...)` | 否 | 指定初始化 |
| 拷贝构造 | `ClassName(const ClassName&)` | 未定义时(浅拷贝) | 复制资源 |
| 移动构造(C++11) | `ClassName(ClassName&&)` | 满足条件时 | 转移资源所有权 |
| 委托构造(C++11) | `ClassName() : ClassName(0,0) {}` | 否 | 复用同类其他构造 |
| 继承构造(C++11) | `using Base::Base;` | 否 | 透传基类构造 |

## 一、默认构造(Default Constructor)

### 三种触发"无默认构造"的情况

```cpp
class A {
public:
    A(int x) {}  // 自定义参数构造
};

A a;          // 错误!无默认构造
A a(1);       // OK
A a{};        // C++11 值初始化,仍需默认构造
A* p = new A; // 错误!
```

### Rule of Zero / Rule of Five / Rule of Three

**Rule of Zero**(推荐):不定义任何构造/析构/赋值操作,让编译器自动生成。前提是成员变量都是 RAII 类型(`std::string`、`std::vector`、`std::unique_ptr` 等)。

**Rule of Five**(需要管理资源时):如果定义了以下**任一**五个函数,通常需要定义全部五个:
- 析构函数
- 拷贝构造
- 拷贝赋值
- 移动构造
- 移动赋值

**Rule of Three**(C++11 前的旧规则):拷贝构造、拷贝赋值、析构函数三选一定义,其余两个也要定义。

### `= default` 与 `= delete`

```cpp
class NonCopyable {
public:
    NonCopyable() = default;                          // 显式要求编译器生成
    NonCopyable(const NonCopyable&) = delete;         // 显式禁止
    NonCopyable& operator=(const NonCopyable&) = delete;
};
```

## 二、拷贝构造(Copy Constructor)

### 签名必须是 `const T&` 而不是 `T`

```cpp
class T {
public:
    T(T other) { ... }        // 错误!无限递归
    T(const T& other) { ... } // 正确
};
```

- `T other` 是值传递,调用拷贝构造时又要拷贝 `other`,无限递归直到栈溢出
- `const T&` 是引用,零拷贝
- `const` 让常量对象和临时对象(`T{}` 创建的右值)都能传进来

### 深拷贝 vs 浅拷贝

```cpp
class Shallow {
    char* buf_;
public:
    Shallow(const Shallow& s) : buf_(s.buf_) {}  // 浅拷贝:共享同一内存
};
// s1.buf_ == s2.buf_,析构时 double free!

class Deep {
    char* buf_;
public:
    Deep(const Deep& d) : buf_(new char[strlen(d.buf_) + 1]) {
        strcpy(buf_, d.buf_);  // 深拷贝:独立内存
    }
};
```

**最佳实践**:用 `std::string`、`std::vector` 替代裸指针/缓冲区,深浅拷贝由 RAII 类型自动处理。

## 三、移动构造(Move Constructor,C++11)

### 性能提升的本质

```cpp
// 拷贝构造:O(n) 时间和空间
std::vector<int> v1 = {1,2,3,4,5};
std::vector<int> v2 = v1;  // 复制 5 个 int

// 移动构造:O(1) 时间和空间
std::vector<int> v3 = std::move(v1);  // 偷指针,v1 变空
```

编译器生成的移动构造是**浅拷贝 + 置空原对象**——把原对象的内部指针"据为己有",原对象进入"有效但未指定"状态(析构安全,但不能读值)。

### 正确实现

```cpp
class Buffer {
    char* data_;
    size_t size_;
public:
    // 移动构造:不分配,直接接管
    Buffer(Buffer&& other) noexcept
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;
        other.size_ = 0;
    }
    
    // 移动赋值:先释放自己的,再接管
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;
            data_ = other.data_;
            size_ = other.size_;
            other.data_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }
};
```

**关键点**:
- 参数是 `T&&`(右值引用)
- 必须 `noexcept`,否则 `std::vector` 扩容时为强异常安全会退化为拷贝
- 移动后必须把原对象置为"有效但未指定"状态(析构安全)

### 编译器自动生成条件

满足以下所有条件时自动生成:
- 没有用户声明的拷贝构造
- 没有用户声明的拷贝赋值
- 没有用户声明的移动赋值
- 没有用户声明的析构函数

### `std::move` 是什么

`std::move` **不移动任何东西**——它只是把左值强制转换为右值引用,触发移动语义:

```cpp
std::string s = "hello";
std::string s2 = std::move(s);  // s 变成空字符串,s2 接管
```

## 四、委托构造(Delegating Constructor,C++11)

### 解决构造函数代码重复

```cpp
class Point {
    int x_, y_;
public:
    // 主构造:实际干活
    Point(int x, int y) : x_(x), y_(y) {
        // 复杂初始化逻辑
    }
    
    // 委托:转给主构造
    Point() : Point(0, 0) {}           // 默认
    Point(int x) : Point(x, 0) {}      // 单参
};
```

### 限制

- 委托构造不能同时是目标构造(不能自己调自己)
- 委托构造的目标构造是唯一真正初始化成员的函数
- 初始化列表中**只能**有委托调用,不能混用其他成员初始化

## 五、继承构造(Inheriting Constructor,C++11)

### 一行代码透传所有基类构造

```cpp
class Base {
public:
    Base(int x) { std::cout << x; }
    Base(int x, double y) { std::cout << x << y; }
};

class Derived : public Base {
public:
    using Base::Base;  // 继承基类的(int)和(int, double)构造
};

Derived d(42);         // 调用 Base::Base(int)
Derived d2(1, 3.14);   // 调用 Base::Base(int, double)
```

### 限制

- 派生类**不能**新增成员?——错,可以新增,只需自己负责初始化:

```cpp
class Derived : public Base {
    int extra_;
public:
    using Base::Base;
    Derived(int x, int extra) : Base(x), extra_(extra) {}  // 自定义需显式初始化基类
};
```

- 继承的构造函数的 `default` 版本不会保留基类的属性(如 `[C++11 新特性](./c++11-新特性.md)`)

## 六、面试高频追问

**Q1: 为什么需要移动构造?**
避免深拷贝临时对象。`std::vector` 扩容时,移动构造只转移内部指针(三个指针 + size + capacity),拷贝构造要逐元素构造新对象。代价从 O(n) 降到 O(1)。`std::string`、`std::unique_ptr`、`std::thread` 都有非平凡析构,移动语义带来数量级性能提升。

**Q2: 拷贝构造参数为什么是 `const T&`?**
- `T&`:非 const 引用,不能接受右值(如 `T{}`、函数返回值)
- `T`:值传递,调用拷贝构造时要拷贝实参,无限递归
- `const T&`:可以接受左值、右值、常量,**且不会无限递归**(引用本身不触发拷贝)

**Q3: 什么情况下编译器不自动生成默认构造?**
只要手动定义了**任一**构造函数,编译器就不再自动生成默认构造。`A(int x) {}` 后,`A a;` 编译失败。`A a{};`(值初始化)也要求默认构造存在。**修复**:用 `= default` 显式要求编译器生成。

**Q4: `std::move` 的本质是什么?**
`std::move(x)` 等价于 `static_cast<T&&>(x)`——**只是类型转换**,不移动任何数据。它告诉编译器"我把这个左值当作右值用",从而触发移动构造/移动赋值。真正的"移动"发生在被赋值的对象的构造/赋值函数中。

**Q5: Rule of Five 的实际意义?**
如果你定义了析构(说明类管理资源),大概率需要定义拷贝和移动(因为默认的浅拷贝会 double free,移动后源对象指针未置空会二次释放)。**例外**:用 RAII 成员(`std::unique_ptr`、`std::string`)时,所有资源管理由 RAII 处理,不需要自定义——这就是 Rule of Zero。
