# 多态

多态（Polymorphism）是 [C++ 学习路线(2026)](./c++-学习路线2026.md) 中面试考察最深的特性，指**同一接口在不同对象上表现出不同行为**。C++ 区分两种实现路径：编译时多态（静态绑定）和运行时多态（动态绑定），前者零开销、后者以 [虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md) 表为代价换取灵活性。

## 定义与核心思想

多态的本质是"调用方写一份代码，运行时/编译时自动选择正确实现"。它解决的核心问题是**类型抽象**——当一个容器、操作或接口需要处理一族相关但不同的类型时，多态让代码不需要写满 `if-else` 或 `switch`。

## 关键原理

### 编译时多态 vs 运行时多态

| 维度 | 编译时多态（静态绑定） | 运行时多态（动态绑定） |
|---|---|---|
| 实现机制 | [重载 vs 重写](./重载-vs-重写.md)、运算符重载、模板 | [虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md) + [继承](./继承.md) |
| 绑定时机 | 编译期 | 运行期（通过 vtable） |
| 性能开销 | 零 | 一次间接寻址 + 可能的 cache miss |
| 类型必须已知 | 是 | 否（基类指针/引用可指向派生类） |
| 典型场景 | 类型已知但行为不同 | 类型运行时才确定 |

**重要**：编译时多态不能完全替代运行时多态——它的前提是"类型在编译期已知"。如果类型是运行时才确定的（如工厂模式返回基类指针），编译时多态无能为力。

### 运行时多态的底层机制：vtable 与 vptr

运行时多态由两个数据结构配合实现：

1. **vtable（虚函数表）**：类级别，每个含虚函数的类编译器生成一张表，存放该类所有虚函数的地址。同类所有对象共享同一张表。
2. **vptr（虚指针）**：对象级别，每个对象内部有一个隐藏指针，指向所属类的 vtable。对象构造时由编译器代码自动设置。

调用过程：

```cpp
Animal* a = new Dog();
a->speak();
```

编译器生成的伪代码：

```
1. 取 a 指向对象的 vptr
2. 在 vptr 指向的 vtable 中找到 speak 的偏移
3. 调用该地址的函数
```

这就是为什么"基类指针指向派生类，调用的却是派生类的实现"——运行期通过 vptr/vtable 完成动态分发。

### 无虚函数的"运行时多态"

[虚函数 vs 纯虚函数](./虚函数-vs-纯虚函数.md) 不是实现运行时多态的唯一路径。常见替代方案：

| 方案 | 原理 | 适用场景 |
|---|---|---|
| 函数指针成员 | 类内存储函数指针，调用时解引用 | 单一行为可变的简单对象 |
| [仿函数 vs Lambda 性能](./仿函数-vs-lambda-性能.md) + 策略模式 | [C++ 多态的实现机制](./c++-多态的实现机制.md) 包装可调用对象 | 行为可灵活替换的场景 |
| [C++ 多态的实现机制](./c++-多态的实现机制.md)（奇异递归模板） | 派生类把自身作为模板参数传给基类 | 零开销的"静态多态" |
| [C++ 多态的实现机制](./c++-多态的实现机制.md) | 抹去具体类型信息，保留行为接口 | `std::function`、`std::any` 等标准库设施 |

`std::function` 的实现是"类型擦除 + 虚函数表"的混合——外部看无虚函数，内部其实用了 vtable。**"无虚函数"不等于"无运行时开销"**，只是把虚函数表的位置从"类内"挪到了"包装器内"。

## 应用与实例

### 运行时多态的最小示例

```cpp
#include <memory>
#include <vector>

class Animal {
public:
  virtual std::string speak() const = 0;
  virtual ~Animal() = default;
};

class Dog : public Animal {
public:
  std::string speak() const override { return "Woof"; }
};

class Cat : public Animal {
public:
  std::string speak() const override { return "Meow"; }
};

void chorus(const std::vector<std::unique_ptr<Animal>>& zoo) {
  for (const auto& a : zoo) {
    std::cout << a->speak() << "\n";  // 运行时多态
  }
}
```

`chorus` 写一次，就能处理 `Dog` / `Cat` / 任何未来的 `Animal` 派生类。

### 编译时多态的最小示例（模板）

```cpp
template <typename Shape>
double area(const Shape& s) {
  return s.area();
}

struct Circle { double r; double area() const { return 3.14 * r * r; } };
struct Square { double side; double area() const { return side * side; } };

area(Circle{1.0});  // 编译期确定调用哪个 area
area(Square{2.0});
```

无虚函数、无 vtable，零开销，但 `area` 函数对每种类型都会生成一份实例化代码（代码膨胀）。

### 典型使用场景

- **运行时多态**：框架扩展点、抽象工厂、Strategy / State / Observer 等行为模式
- **编译时多态**：数值计算、容器、STL 风格的泛型库、嵌入式等性能敏感场景
- **CRTP**：编译器插件、性能关键的多态调用
