# C++ 完美转发

## 核心结论

完美转发是在函数模板中把参数**以原始值类别**(左值/右值)**无损地传递给另一个函数**。靠三个机制配合:**万能引用**接收(同时接受左值/右值)、**引用折叠**自动推导、`std::forward<T>` 有条件恢复。效果:传进来是左值就按左值转发,传进来是右值就按右值转发,移动语义不会丢失。

## 一、解决什么问题

### 不完美转发的浪费

```cpp
template<typename T, typename Arg>
T* create(Arg arg) {              // arg 在函数体内永远是左值
    return new T(arg);             // 永远走拷贝,移动语义丢失
}

std::string s = "hello";
create<std::string>(s);                    // 左值 → 拷贝
create<std::string>(std::string("temp"));  // 右值 → 仍拷贝(浪费!)
create<std::string>(std::move(s));         // 右值 → 仍拷贝(浪费!)
```

### 完美转发的修复

```cpp
template<typename T, typename Arg>
T* create(Arg&& arg) {                    // 万能引用
    return new T(std::forward<Arg>(arg)); // 完美转发
}

create<std::string>(s);                    // 转发为左值 → 拷贝
create<std::string>(std::string("temp"));  // 转发为右值 → 移动
create<std::string>(std::move(s));         // 转发为右值 → 移动
```

## 二、万能引用 + 引用折叠

### 万能引用只能出现在两个上下文

```cpp
template<typename T>
void f(T&& arg);     // 1. 模板参数推导中的 T&&
                      //    (具体类型的 T,非 typename T::value_type&&)

auto&& x = ...;      // 2. auto 推导中的 auto&&
```

**对比**:

```cpp
template<typename T>
void f(T&& arg);            // 万能引用:取决于 T 的推导

void g(std::string&& s);    // 普通右值引用:T 已固定
```

### 引用折叠规则

| 组合 | 折叠结果 | 触发场景 |
|------|---------|---------|
| `T& &` | `T&` | `T = int&` 推导后 `T& &&` |
| `T& &&` | `T&` | 同上 |
| `T&& &` | `T&` | `T = int` 推导后 `int&& &` |
| `T&& &&` | `T&&` | `T = int` 直接右值,`T&& = int&&` |

**口诀:有左值引用就折叠成左值引用**,只有"双右值"才保持右值。

### 推导示例

```cpp
template<typename T>
void f(T&& arg);

int x = 42;
f(x);     // T = int&,arg 类型 = int& && = int&(左值引用)
f(42);    // T = int,arg 类型 = int&&(右值引用)
```

## 三、`std::forward` 的工作原理

### 条件转换

```cpp
template<typename T>
T&& forward(std::remove_reference_t<T>& t) noexcept {
    return static_cast<T&&>(t);
}

// T = int&(左值):返回 int&  (保持左值)
// T = int(右值):返回 int&& (转回右值)
```

| 调用者传的 | T 推导 | forward 返回 | 触发 |
|-----------|--------|-------------|------|
| 左值 | `int&` | `int&` | 拷贝 |
| 右值 | `int` | `int&&` | 移动 |

### 关键洞察

- `forward` 在编译期根据 `T` 的推导结果**有条件**转换
- `move` 是**无条件**转右值(不管 `T` 是什么)
- 模板转发场景用 `move` 会破坏调用者的预期(左值也被强制转右值)

## 四、典型应用场景

### 1. `make_unique` / `make_shared` 的实现

```cpp
template<typename T, typename... Args>
std::unique_ptr<T> make_unique(Args&&... args) {
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
    //                                           ↑ 完美转发给 T 的构造函数
}

// 用户可以传入任意左值/右值,触发对应的拷贝/移动
auto p = make_unique<std::string>("hello");        // const char* → string(const char*)
auto p2 = make_unique<std::string>(std::move(s));  // 移动构造
```

### 2. `emplace_back` 的实现

```cpp
template<typename T>
class vector {
    template<typename... Args>
    void emplace_back(Args&&... args) {
        // 在 vector 内部直接构造元素,避免临时对象
        new (data_ + size_) T(std::forward<Args>(args)...);
    }
};

// 等价于 push_back,跳过临时对象
v.emplace_back("hello", 42);  // 直接调 Widget(string, int) 构造函数
```

### 3. 包装函数

```cpp
template<typename Func, typename... Args>
auto wrapper(Func&& f, Args&&... args) -> decltype(f(std::forward<Args>(args)...)) {
    return std::forward<Func>(f)(std::forward<Args>(args)...);
    // ↑ 转发函数对象本身
    // ↑ 转发所有参数
}
```

## 五、可变参数模板 + 完美转发

### 参数包展开

```cpp
template<typename Func, typename... Args>
auto call(Func&& f, Args&&... args) {
    return std::forward<Func>(f)(std::forward<Args>(args)...);
    //                                  ↑ 展开:每个参数独立推导、独立转发
}

void f(int, std::string, double);
call(f, 42, std::string("hi"), 3.14);
// 42: int 左值 → 转发左值
// "hi": 临时 string 右值 → 转发右值(移动构造)
// 3.14: 临时 double 右值 → 转发右值
```

### 在 `emplace` 中的体现

```cpp
v.emplace_back(1, 2, 3);   // 转发 3 个参数
v.emplace_back("hello");   // 转发 1 个参数
// 内部展开:new T(std::forward<int>(1), std::forward<int>(2), std::forward<int>(3))
```

## 六、完美转发失败的场景

### 1. 传递花括号初始化列表

```cpp
template<typename T>
void f(T&& arg);

f({1, 2, 3});  // 错误!T 无法推导为 initializer_list<int>
```

**修复**:明确指定类型或用 `initializer_list<T>` 参数。

### 2. 传递 `0` 或 `NULL` 作为空指针

```cpp
template<typename T>
void f(T&& arg);

f(0);     // T 推导为 int,不是指针!
f(NULL);  // T 推导为 long(取决于 NULL 的定义)
```

**修复**:用 `nullptr`,类型是 `std::nullptr_t`,能正确推导。

### 3. 传递重载函数名

```cpp
template<typename T>
void f(T&& arg);

void g(int);
void g(double);

f(g);  // 错误!T 不知道推导为哪个 g
```

**修复**:显式指定:`f(static_cast<void(*)(int)>(g))`。

### 4. 位域

```cpp
struct S { int x : 4; };
S s;
f(s.x);  // 错误!位域不能绑定到引用
```

**修复**:拷贝到中间变量再传:`int tmp = s.x; f(tmp);`

## 七、`move` vs `forward` 决策

| 场景 | 用什么 | 原因 |
|------|--------|------|
| 明确"我不要这个对象了" | `std::move` | 无条件转右值 |
| 模板中转发参数 | `std::forward` | 保持原始值类别 |
| 容器 `push_back` 临时对象 | `std::move` | 明确转移所有权 |
| `emplace_back` 实现 | `std::forward` | 不知道调用者传左值还是右值 |

**核心区别**:
- `move`:**无条件**转右值,语义"我不要了,你拿走"
- `forward`:**有条件**保持原值类别,语义"你给我什么,我原样传下去"

## 八、面试高频追问

**Q1: `forward` 和 `move` 有什么区别?**
- `std::move`:**无条件**转右值(不管传什么,出去都是右值)
- `std::forward`:**有条件**保持原值类别(根据模板参数 `T` 推导结果)
- `move` 用在"明确要转移所有权"场景,`forward` 用在"模板转发参数"场景
- 两者本质都是 `static_cast`,零运行时开销

**Q2: 完美转发最典型的应用场景?**
- 工厂函数(`make_unique`/`make_shared`)
- 容器 `emplace` 系列(`emplace_back` 把参数转发给元素构造函数)
- 回调包装器(把参数原样转发给回调)
- 共同点:**中间层不知道目标函数需要左值还是右值**

**Q3: 完美转发什么时候会失败?**
- 传递 `{1,2,3}` 花括号初始化列表(`T` 无法推导为 `initializer_list`)
- 传递 `0`/`NULL` 当空指针(`T` 推导为 `int`)
- 传递重载函数名(`T` 不知道推导为哪个重载)
- 传递位域(不能绑定引用)

**Q4: 不用 `forward` 直接传 `arg` 会怎样?**
`arg` 在函数体内有名字,**永远是左值**。直接传给目标函数会走左值版本(拷贝),右值的移动语义丢失。这就是"不完美转发"——所有参数退化成左值。

**Q5: 引用折叠规则是什么?**
**有左值引用就折叠成左值引用**。`T& &`/`T& &&`/`T&& &` 全部折叠成 `T&`,只有 `T&& &&` 保持 `T&&`。这是万能引用和 `std::forward` 工作的基础。
