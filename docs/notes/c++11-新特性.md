# C++11 新特性全景

## 核心结论

C++11 是**现代 C++ 的分水岭**,所有新特性围绕三个核心目标展开:

| 目标        | 代表特性                                     | 解决的核心问题             |
| --------- | ---------------------------------------- | ------------------- |
| **性能优化**  | 右值引用、移动语义、`constexpr`                    | 减少不必要拷贝,把能编译期做的事提前做 |
| **安全性提升** | 智能指针、`nullptr`、强类型枚举、`static_assert`     | 从语言层面堵住 C++ 的经典坑    |
| **表达力增强** | `auto`、Lambda、范围 for、初始化列表、变参模板          | 让代码更简洁,意图更明确        |
| **并发支持**  | `std::thread`、`std::mutex`、`std::atomic` | 跨平台标准多线程库           |

## 一、性能优化:右值引用与移动语义

### 问题的根源

C++11 之前,函数返回大对象、容器插入临时对象,都要经历一次深拷贝:

```cpp
std::vector<std::string> v;
std::string s = "hello, world!";
v.push_back(s);               // 拷贝构造:分配新内存,复制字符
v.push_back("temporary");    // 拷贝构造:从临时对象复制
v.push_back(std::string("x") + std::string("y"));  // 中间有临时对象,反复拷贝
```

### 右值引用的解法

```cpp
template<class T>
class vector {
    void push_back(const T& value);  // 拷贝
    void push_back(T&& value);        // C++11:移动
};

std::string s = "hello";
v.push_back(s);                      // 调拷贝版本
v.push_back(std::move(s));           // 显式转右值,调移动版本
v.push_back("temp");                 // 临时对象,自动选移动版本
```

移动构造的本质:**浅拷贝 + 置空原对象**。对 `std::string` 来说,只交换三个指针(size/capacity/ptr),O(1) 完成,而不是 O(n) 复制字符。

### `constexpr` 的方向

`constexpr` 把能在编译期算出来的值提前算好:

```cpp
constexpr int factorial(int n) {  // 编译期函数
    return n <= 1 ? 1 : n * factorial(n - 1);
}
int arr[factorial(5)];  // 编译期算 120,分配 120 元素数组
```

C++11 限制单语句,C++14 放宽,C++17 允许 `if`/`for`,C++20 允许虚函数调用。

## 二、安全性提升:从语言堵住经典坑

### 智能指针家族

```cpp
// 独占所有权,零开销(C++11 默认选择)
std::unique_ptr<T> up = std::make_unique<T>();

// 共享所有权,有引用计数开销
std::shared_ptr<T> sp = std::make_shared<T>();

// 观察但不拥有,解决循环引用
std::weak_ptr<T> wp = sp;
```

**Rule of Zero 时代**:不写 `new`/`delete`,不写析构/拷贝/移动。智能指针和 STL 容器是 RAII,自动管理生命周期。

### `nullptr` 替代 `NULL`

```cpp
void f(int);
void f(int*);
f(NULL);      // C:调 f(int),因为 NULL 是 0;可能不是你想要的
f(nullptr);   // C++11:明确调 f(int*),无歧义
```

`nullptr` 的类型是 `std::nullptr_t`,可隐式转换到任意指针类型,但**不**能转换到整数。

### 强类型枚举(`enum class`)

```cpp
// 旧枚举的问题
enum Color { RED, GREEN };
enum Status { OK, ERROR };
int x = RED;        // 隐式转 int,污染命名空间
RED == 0;           // 居然为真

// 强类型枚举
enum class Color { Red, Green };
enum class Status { Ok, Error };
Color c = Color::Red;   // 必须加作用域
int x = c;              // 错误!不隐式转 int
if (c == Color::Red) {} // 必须显式比较
```

### `static_assert` 编译期断言

```cpp
static_assert(sizeof(int) == 4, "Requires 32-bit int");

template<class T>
void process() {
    static_assert(std::is_integral_v<T>, "T must be integral");
    // ...
}
```

错误在编译期暴露,运行期零开销。比 `assert()` 宏(运行期)和 `#if` 预处理(功能受限)都好。

## 三、表达力增强:auto / Lambda / 范围 for

### `auto` 简化类型声明

```cpp
// 旧 C++:复杂迭代器类型
std::map<std::string, std::vector<int>>::iterator it = m.begin();

// C++11:auto 推导
auto it = m.begin();

// 用于 lambda
auto cmp = [](int a, int b) { return a < b; };
std::sort(v.begin(), v.end(), cmp);
```

**注意**:`auto` 推导会**忽略顶层 const 和引用**:

```cpp
const int x = 42;
auto y = x;       // y 是 int,不是 const int!
auto& z = x;      // z 是 const int&(显式加 & 保留 const)
```

`decltype` 保留完整类型,适合模板元编程(见 [auto vs decltype](/notes/auto-vs-decltype.html) 文章)。

### Lambda 表达式

```cpp
// 旧 C++:仿函数(functor)需要写整个类
struct GreaterThan {
    int threshold;
    GreaterThan(int t) : threshold(t) {}
    bool operator()(int x) const { return x > threshold; }
};
auto pred = GreaterThan(10);

// C++11:lambda 一行搞定
auto pred = [](int x) { return x > 10; };

// 带捕获
int threshold = 10;
auto pred1 = [threshold](int x) { return x > threshold; };  // 按值捕获
auto pred2 = [&threshold](int x) { return x > threshold; }; // 按引用捕获
auto pred3 = [=](int x) { return x > threshold; };  // 全部按值
auto pred4 = [&](int x) { return x > threshold; };  // 全部按引用
```

**最佳实践**:显式列出要捕获的变量(意图清晰,避免意外):

```cpp
auto callback = [threshold, &result](int x) {
    result = x * threshold;
};
```

### 范围 for 循环

```cpp
// 旧 C++:迭代器三件套
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it;
}

// C++11 范围 for
for (const auto& elem : v) {
    std::cout << elem;
}
```

**陷阱**:范围 for 遍历时**不要修改容器**(增删元素),迭代器失效会导致未定义行为。

### 统一初始化 `{}`

```cpp
int x{42};                       // 直接初始化
std::vector<int> v{1, 2, 3};     // 初始化列表
std::map<std::string, int> m{{"a", 1}, {"b", 2}};  // 嵌套初始化
MyClass obj{arg1, arg2};         // 调用对应构造函数
```

`{}` 禁止窄化转换:`int x{3.14};` 编译错误,`int x(3.14);` 只是警告。

### 变参模板

```cpp
// 任意数量、任意类型的参数
template<typename... Args>
void print(Args... args) {
    (std::cout << ... << args) << '\n';  // C++17 折叠表达式
}

print(1, "hello", 3.14, 'x');  // OK
```

## 四、并发支持:标准多线程库

C++11 之前多线程依赖平台 API:

```cpp
// POSIX:pthread_create
// Windows:CreateThread
// 两套 API,代码不通用
```

C++11 引入标准线程库:

```cpp
#include <thread>
#include <mutex>
#include <condition_variable>
#include <atomic>
#include <future>

std::mutex m;
std::atomic<int> counter{0};

void worker() {
    std::lock_guard<std::mutex> lock(m);
    counter++;
}

std::thread t1(worker);
std::thread t2(worker);
t1.join();
t2.join();
```

跨平台、性能与平台原生 API 接近(底层 pthread/Win32 thread 封装)。

## 五、其他重要特性

### `override` 和 `final`

```cpp
class Derived : public Base {
    void f() override;  // 显式标记重写虚函数,编译期检查
};
class Sealed final : public Base {};  // 禁止再被继承
```

- `override`:编译器检查基类是否有同名虚函数,防止拼写错误
- `final`:`class` 标记禁止继承,`virtual func` 标记禁止重写

### `default` 和 `delete`

```cpp
class NonCopyable {
public:
    NonCopyable() = default;                      // 显式让编译器生成
    NonCopyable(const NonCopyable&) = delete;    // 显式禁止
};
```

### `std::array` / `std::tuple` / `std::function`

```cpp
std::array<int, 5> arr = {1,2,3,4,5};   // 固定大小数组,栈分配
std::tuple<int, double, std::string> t = {42, 3.14, "hi"};
auto [i, d, s] = t;                      // C++17 结构化绑定

std::function<int(int)> f = [](int x) { return x * 2; };  // 类型擦除的函数包装
```

## 六、面试高频追问

**Q1: `auto` 和 `decltype` 的区别?**
- `auto`:用于变量声明,从初始化表达式推导类型,**忽略顶层 const 和引用**(`auto x = 42;` → `int`)
- `decltype`:用于获取表达式的精确类型,**保留 const 和引用**(`decltype((x)) y = x;` → `int&`)
- 典型场景:`auto` 简化日常代码,`decltype` 用于模板返回值推导等需要精确类型的场景

**Q2: 右值引用为什么能提升性能?**
让编译器区分"还要继续用的对象"和"马上销毁的临时对象"。对临时对象,移动构造直接接管内部资源(如 `vector` 的指针),而不是分配新内存逐元素拷贝。代价从 O(n) 降到 O(1)。`std::move(x)` 把左值强制转右值引用,显式触发移动。

**Q3: 三个智能指针怎么选?**
- **`unique_ptr`**:独占所有权,零额外开销(C++11 默认选择)
- **`shared_ptr`**:共享所有权,有引用计数开销(原子操作,2x 慢)
- **`weak_ptr`**:配合 `shared_ptr` 解决循环引用——它观察但不拥有,不增加引用计数

**Q4: Lambda 捕获列表的讲究?**
- `[=]` 按值捕获:安全(独立副本),但有拷贝开销
- `[&]` 按引用捕获:零开销,但 Lambda 比被捕获变量活得久就悬空
- **最佳实践**:显式列出要捕获的变量(意图清晰),避免 `[=]`/`[&]` 的隐式捕获

**Q5: `nullptr` 相比 `NULL` 解决了什么问题?**
`NULL` 本质是整数 0(`#define NULL 0`),会触发整型重载。`nullptr` 类型是 `std::nullptr_t`,可隐式转任意指针,**不能**转整数。`f(NULL)` 调 `f(int)`,`f(nullptr)` 调 `f(int*)`,意图明确。
